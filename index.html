Banking Website Design 2021

프로젝트 개요

사용자 친화적인 웹디자인을 기반으로 한 가상의 모바일 은행 웹사이트 프로젝트입니다. 직관적인 인터페이스와 효율적인 기능을 통해 사용자가 쉽게 접근하고 원하는 서비스를 이용할 수 있도록 디자인 되었습니다.

프로젝트 인원: 개인 프로젝트

사용 도구: Figma

Project Overview

This project is a virtual mobile banking website based on user-friendly web design. It was designed with a focus on user experience.

Team Size: 1 person (individually created)

Tools Used: Figma

프리랜서, 스타트업 창업자, 소규모 사업자를 주요 타깃으로 한 가상의 디지털 뱅킹 플랫폼을 기획한 작업입니다. 전통적인 금융 서비스의 딱딱함을 탈피하고, 젊고 활기찬 이미지를 전달하기 위해 생동감 있는 색감과 모던한 디자인 요소를 적극 활용했습니다. 디자인 컨셉은 단순히 기능적 요구를 넘어, 사용자에게 금융 서비스를 보다 친근하고 접근하기 쉽게 전달하는 데 초점을 맞췄습니다. 이를 통해 기존의 금융 시스템에서 느낄 수 있는 부담감을 줄이고, 새로운 경험을 선사하고자 하였습니다.

Overall Concept: This project was conceived as an innovative digital banking platform targeting freelancers, startup founders, and small business owners. The design breaks away from the rigid image of traditional banking by incorporating a fresh, vibrant aesthetic. By utilizing a bold color palette and modern design elements, the concept aims to provide an approachable and engaging financial service experience.

은행 특징 및 카드 종류별 혜택 등 핵심 기능을 직관적으로 배치하여 사용자가 쉽게 원하는 서비스를 찾을 수 있도록 구성하였습니다. 텍스트와 아이콘의 적절한 조합으로 가독성과 정보 전달력을 극대화하여, 중요한 요소는 자연스럽게 눈에 띄도록 디자인하였습니다.

Detailed Features: The layout clearly organizes key functionalities such as account creation, fund management, and money transfers, mirroring a live interface. The strategic balance of text and icons ensures that users can quickly understand the purpose of each feature. Critical elements are highlighted to capture user attention, creating a seamless and intuitive navigation experience throughout the platform.

CodeEasy 2024

프로젝트 개요

저시력자를 위한 코딩 알고리즘 학습 웹사이트입니다. 주 사용자인 저시력자를 고려하여 컬러감을 강조하고, 음성 TTS 기능 등 접근성을 최우선으로 설계되었습니다.

프로젝트 인원: 프론트엔드 팀 3명, 백엔드 팀 3명

프로젝트 기여도: 로고, 메인 화면 페이지, 알고리즘 문제 페이지, 채팅 기능 구현

사용 도구: Figma, React

Project Overview

This website is designed for individuals with low vision to learn coding algorithms. It emphasizes color schemes that are accessible for visually impaired users and prioritizes accessibility features such as text-to-speech (TTS) functionality.

Team Size: 3 frontend developers, 3 backend developers

Contributions: Implemented the main page, algorithm problem page, and chat functionality.

Tools Used: Figma, React

프로젝트 기획의도
코드이지(CodeEasy)는 저시력자도 쉽게 접근하여 알고리즘 문제를 풀고 학습할 수 있는 웹사이트입니다. 04.20 장애인의 날을 맞아, 웹 사용 시 불편함을 겪는 사용자들을 위해 음성 TTS, 화면 확대, 고대비 테마, 큰 글씨 등 다양한 접근성 기능을 적용하였습니다.


CodeEasy is a website designed for individuals with low vision to easily access and learn coding algorithms. In honor of the Day of Persons with Disabilities on April 20, various accessibility features such as voice TTS, screen magnification, high-contrast themes, and large text have been implemented to assist users who face difficulties while using the web.


핵심 기능 & UI/UX 설계
테마 설정 페이지에서 다크, 라이트, 고대비 모드를 자유롭게 전환할 수 있어, 사용자가 자신의 시각적 요구에 맞게 인터페이스를 조절할 수 있습니다. 또한, 음성 TTS 기능을 통해 메뉴, 문제, 정답, 풀이 등의 내용을 음성으로 안내하며, 화면 확대 기능으로 콘텐츠를 더욱 쉽게 확인할 수 있도록 설계하였습니다.

Development & Interaction:
프론트엔드에서는 React와 TypeScript, SCSS를 활용해 모듈화된 컴포넌트를 개발하고, Recoil을 통한 효율적인 상태관리를 구현했습니다. 각 팀원은 메인 홈, 알고리즘 문제 페이지, 채팅, 로그인/회원가입, 마이 페이지, 테마 설정, 문제 IDE 등 다양한 영역에서 핵심 기능을 맡아 프로젝트의 완성도를 높였습니다.

UX Focus:
사용자 경험을 최우선으로 고려하여, 누구나 쉽게 접근하고 사용할 수 있는 인터페이스를 제공하는 동시에, 시각적 접근성 개선을 통한 몰입감 있는 학습 환경을 구현했습니다.


Key Features & UI/UX Design:
The theme settings page allows users to freely switch between dark, light, and high-contrast modes, enabling them to adjust the interface according to their visual needs. Additionally, the voice TTS feature provides audio guidance for menus, problems, answers, and solutions, while the screen magnification feature is designed to make content easier to view.

Development & Interaction:
The frontend was developed using React, TypeScript, and SCSS to create modular components, and efficient state management was implemented using Recoil. Each team member was responsible for key features across various areas, including the main home, algorithm problem pages, chat, login/signup, my page, theme settings, and problem IDE, enhancing the overall quality of the project.

UX Focus:
User experience was prioritized to provide an interface that is easily accessible and usable by everyone, while also creating an immersive learning environment through improvements in visual accessibility.


Design & Development

GroovyMap 2024

프로젝트 개요

음악, 댄스 등 각 공연예술 분야의 사람들을 위한 커뮤니티 웹앱으로, 지도를 활용한 공연 연습, 거리 공연장 찾기 및 커뮤니티 기능이 합쳐진 사이트

프로젝트 인원: 프론트 3명, 백엔드 3명

프로젝트 기여도: 전체적인 UI/UX 설계, 메인 화면, 로그인 회원 가능 페이지 디자인 및 기능 구현, 공연 장소 및 연습 장소 구현, 지도 API 사용

사용 도구: Figma, Next.js

Project Overview

This is a community web app for people in various performing arts fields such as music and dance, combining features for rehearsal, finding street performance venues, and community functions.

Team Size: 3 frontend developers, 3 backend developers

Contributions: Overall UI/UX design, main screen, design and implementation of login member pages, implementation of performance and rehearsal venues, use of map API.

Tools Used: Figma, Next.js

프로젝트 기획의도
공연 예술가들이 정보를 공유하고, 협업할 팀원을 찾고, 자신의 공연을 홍보할 수 있는 온라인 커뮤니티를 제공하기 위해 기획되었습니다. 기존의 SNS나 일반 커뮤니티 플랫폼에서는 공연 예술가들이 네트워킹하거나 공연 장소를 찾기 어려운 경우가 많아, 보다 특화된 커뮤니티가 필요하다고 판단했습니다.

기본적인 CRUD 기능을 넘어서, 카카오맵 API를 활용한 공연 및 연습 공간 정보 제공, 팀원 모집 기능, 공연 홍보 게시판, AI 이미지 생성 등 다양한 기술을 적용하면서 개발 실력을 쌓을 수 있도록 했습니다.


This project was planned to provide an online community where performing artists can share information, find collaborators, and promote their performances. Existing SNS or general community platforms often make it difficult for performing artists to network or find performance venues, so a more specialized community was deemed necessary.

Beyond basic CRUD functionality, various technologies were applied, including performance and rehearsal space information using KakaoMap API, team recruitment features, performance promotion boards, and AI-generated images, allowing for skill development during the process.



핵심 기능 & UI/UX 설계
UI는 심플하고 직관적인 디자인을 적용하여, 사용자들이 콘텐츠에 집중할 수 있도록 최소화하였습니다. 메인 페이지에서는 각 페이지로 이어지는 버튼과 사이트 소개 화면을 제공하고, 카카오맵 API를 활용해 위치 기반의 검색 기능을 지원합니다. 또한, 프로필 및 팀원 모집 기능을 통해 사용자의 네트워킹을 강화하며, 공연 및 연습장소 게시판을 통해 지도를 보며 위치 정보를 쉽게 찾을 수 있으며, 공연 홍보 및 자유 게시판, 소셜 기능(팔로우, DM)을 통해 사용자 간의 소통을 활성화합니다.

Development & Interaction
프론트엔드 개발에서는 Next.js와 TypeScript를 사용하여 모듈화된 컴포넌트를 개발하고, Tailwind CSS를 활용해 스타일링의 일관성을 유지했습니다. Recoil을 이용해 상태 관리를 효율적으로 처리했으며, 각 팀원은 공연 및 연습 장소 검색, 프로필 관리, 팀원 모집 기능, 공연 홍보 게시판 등 다양한 핵심 기능을 담당하여 프로젝트의 완성도를 높였습니다. 각 기능은 독립적으로 개발되었으며, 원활하게 통합되었습니다.

UX Focus
사용자 경험을 최우선으로 고려하여 누구나 쉽게 접근하고 사용할 수 있는 직관적인 인터페이스를 설계했습니다. 공연 예술가들이 빠르게 필요한 정보를 찾고, 네트워킹하며, 협업할 수 있도록 디자인 요소를 최소화하고, 정보 중심의 레이아웃을 채택했습니다. 또한, 시각적 접근성을 고려하여, 사용자들이 몰입할 수 있는 환경을 만들었으며, 직관적인 검색 및 게시물 업로드 기능을 통해 사용자가 더 효율적으로 플랫폼을 이용할 수 있도록 했습니다.


Key Features & UI/UX Design
The UI is simple and intuitive, reducing distractions so users can focus on content. The main page has buttons to different sections and an introduction to the site. It supports location-based search with KakaoMap API. Profile and team recruitment features help with networking, and venue boards make it easy to find locations on the map. Social features like follow, DM, and promotion boards encourage user interaction.

Development & Interaction
For front-end development, We used Next.js and TypeScript for front-end development to create modular components, with Tailwind CSS for consistent styling. Recoil was used for state management. Each team member worked on key features like venue search, profile management, team recruitment, and promotion boards, ensuring the project was complete. All features were developed separately and integrated smoothly.

UX Focus
We focused on user experience by designing an easy-to-use interface. We minimized design elements and used a content-focused layout, making it easier for artists to find information, network, and collaborate. We improved visual accessibility for an immersive experience and added easy search and post-upload features to improve usability.
